Git Reflog

Git reflog is a command used to track updates made to the tip of branches. It records when the HEAD changes in a Git repository.

Why Use Git Reflog?

  • Recover lost commits
  • Undo a hard reset
  • Find previous branch states

Basic Command

Run the following command to see the reflog history:

git reflog

Example Output

a1b2c3d HEAD@{0}: commit: Added login feature
e4f5g6h HEAD@{1}: commit: Updated README
i7j8k9l HEAD@{2}: checkout: moving from main to feature-branch

Recover a Lost Commit

You can restore a previous state using:

git reset --hard HEAD@{1}

This resets your repository to the state recorded in the reflog.
